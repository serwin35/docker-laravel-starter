# 🚀 Quickstart Guide

Complete step-by-step guide for running Laravel with this starter kit — from zero to a working application.

---

## Requirements

| Tool | Version | Check |
|---|---|---|
| Docker Desktop | 4.x+ | `docker --version` |
| Docker Compose | v2+ | `docker compose version` |
| Git | any | `git --version` |
| GNU Make | any | `make --version` |
| GitHub CLI *(optional)* | 2.x | `gh --version` |

> PHP and Composer are **not required** locally — everything runs inside the container.

---

## Scenario A — New Laravel Project (from scratch)

### 1. Clone the starter kit

```bash
git clone https://github.com/serwin35/docker-laravel-starter.git my-project
cd my-project
```

### 2. Configure your environment

```bash
cp .env.example .env
```

Open `.env` and set your project values:

```ini
APP_NAME=my-project
APP_URL=http://localhost

# Choose your database: mysql or pgsql
DB_CONNECTION=mysql
DB_HOST=mysql          # change to: pgsql if using PostgreSQL
DB_DATABASE=my_project
DB_USERNAME=laravel
DB_PASSWORD=secret
```

### 3. Build the PHP image

```bash
make build ENV=dev DB=mysql
```

### 4. Install Laravel inside the container

```bash
make laravel-new
```

> This runs `composer create-project laravel/laravel` inside the PHP container and copies the files into your working directory. No local PHP needed.

### 5. Generate app key

```bash
make artisan ARGS="key:generate"
```

### 6. Start the containers

```bash
# MySQL (default)
make up DB=mysql

# or PostgreSQL
make up DB=pgsql
```

### 7. Run migrations

```bash
make migrate
```

### 8. Done

```
http://localhost:80  →  Laravel welcome page
```

## Scenario B — Existing Laravel Project

You already have a Laravel project and want to add this Docker stack to it.

### 1. Go to your project directory

```bash
cd /path/to/your/laravel-project
```

### 2. Download the Docker starter files

```bash
# Downloads and extracts only the Docker files (no README, LICENSE, .git)
curl -fsSL https://github.com/serwin35/docker-laravel-starter/archive/refs/heads/main.tar.gz \
  | tar xz --strip-components=1 \
    --exclude='docker-laravel-starter-main/README.md' \
    --exclude='docker-laravel-starter-main/LICENSE' \
    --exclude='docker-laravel-starter-main/.git'
```

Or manually copy from a cloned repo:

```bash
git clone https://github.com/serwin35/docker-laravel-starter.git /tmp/docker-starter
cp -r /tmp/docker-starter/.docker         ./
cp -r /tmp/docker-starter/.github         ./
cp -r /tmp/docker-starter/scripts         ./
cp    /tmp/docker-starter/docker-compose*.yml ./
cp    /tmp/docker-starter/Makefile        ./
cp    /tmp/docker-starter/.dockerignore   ./
cp    /tmp/docker-starter/.env.example    ./.env.docker-example
```

### 3. Merge Docker variables into your `.env`

Open `.env.docker-example` and add the Docker-specific sections to your existing `.env`:

```ini
# ── Docker ────────────────────────────────
APP_PORT=80
UID=1000
GID=1000

# ── Redis ─────────────────────────────────
REDIS_HOST=redis
REDIS_PASSWORD=redissecret
CACHE_STORE=redis
QUEUE_CONNECTION=redis
SESSION_DRIVER=redis

# ── Mail (Mailpit for dev) ─────────────────
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_MAILER=smtp
```

Make sure `DB_HOST` points to the Docker service name:

```ini
DB_HOST=mysql     # or: pgsql — NOT 127.0.0.1
```

### 4. Start

```bash
make up DB=mysql
make migrate
```

## Choosing a Database

The database is activated via Docker profiles:

```bash
# MySQL 8.4 (default)
make up DB=mysql

# PostgreSQL 16
make up DB=pgsql
```

Matching `.env` settings:

```ini
# MySQL
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306

# PostgreSQL
DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
```

> Switching databases requires `make down` followed by `make up DB=pgsql`. Data is not migrated between engines automatically.

## Optional Services

Additional services are activated via Docker profiles.

### Vite — hot-reload frontend

```bash
make npm-dev
# or:
COMPOSE_PROFILES=npm,mysql make up
```

Vite available at `http://localhost:5173`

### MinIO — local S3-compatible storage

```bash
COMPOSE_PROFILES=minio,mysql make up
```

| URL | Description |
|---|---|
| `http://localhost:9001` | Web console (user: minio / pass: minioadmin) |
| `http://localhost:9000` | S3 API endpoint |

Required in your project:
```bash
make composer ARGS="require league/flysystem-aws-s3-v3"
```

See `.env.example` → MinIO section for all required variables.

### Laravel Reverb — WebSocket server

```bash
COMPOSE_PROFILES=reverb,mysql make up
```

WebSocket listening at `http://localhost:8080`

Required:
```bash
make composer ARGS="require laravel/reverb"
make artisan ARGS="reverb:install"
```

### Mailpit — fake SMTP

Starts automatically with every `make up`. Open:
```
http://localhost:8025
```

All outgoing emails from your Laravel app are captured here.

## SSL / HTTPS Locally

```bash
# Generate a self-signed certificate
bash scripts/gen-certs.sh              # for localhost
bash scripts/gen-certs.sh myapp.local  # for a custom domain
```

Then swap the Nginx config in `docker-compose.dev.yml`:

```yaml
nginx:
  volumes:
    - ./.docker/nginx/conf.d/app-ssl.conf:/etc/nginx/conf.d/default.conf:ro
    - ./.docker/nginx/certs:/etc/nginx/certs:ro
  ports:
    - "443:443"
    - "80:80"
```

Update `.env`:
```ini
APP_URL=https://localhost
APP_PORT=443
```

Trust the certificate in macOS:
```bash
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain \
  .docker/nginx/certs/self-signed.crt
```

## Xdebug (Development)

Xdebug 3 is built into the `development` image, configured for port `9003`.

**PhpStorm:**
1. Preferences → PHP → Debug → port: `9003`
2. Click "Start Listening for PHP Debug Connections"
3. Set a breakpoint and open the app in your browser

**VS Code** — `.vscode/launch.json`:
```json
{
  "name": "Listen for Xdebug",
  "type": "php",
  "request": "launch",
  "port": 9003,
  "pathMappings": {
    "/var/www/html": "${workspaceFolder}"
  }
}
```

Control Xdebug mode via `.env`:
```ini
XDEBUG_MODE=debug      # step debugging
XDEBUG_MODE=coverage   # code coverage only
XDEBUG_MODE=off        # disabled (faster startup)
```

## Troubleshooting

### Permission denied on storage/

```bash
make artisan ARGS="storage:link"
docker compose exec php chmod -R 775 storage bootstrap/cache
```

### Port 80 already in use

```ini
# .env
APP_PORT=8080
```

### PHP container won't start

```bash
make logs ARGS=php
# Check that APP_KEY is set in .env
make artisan ARGS="key:generate"
```

### Database connection refused

Make sure `DB_HOST` is the service name (`mysql` or `pgsql`), **not** `127.0.0.1`.
Wait for the database healthcheck to pass:

```bash
make ps   # check STATUS column — should show "healthy"
```
